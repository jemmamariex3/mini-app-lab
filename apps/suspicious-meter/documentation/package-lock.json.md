# File Documentation: `package-lock.json`

## File Name and Path

- File: `package-lock.json`
- Path: `apps/suspicious-meter/package-lock.json`

## Purpose

Records the exact resolved npm dependency tree for the Suspicious Meter app so installs remain reproducible across environments.

## Main Exports

- npm lockfile metadata consumed by npm during installation

## Inputs

- Dependency declarations from `package.json`
- Versions resolved by npm at install time

## Internal State or Managed Data

- Top-level package metadata for the app
- Exact resolved versions for runtime and development dependencies
- Integrity hashes and package source information for installed modules

## Key Logic and Behavior

- Locks dependency resolution to specific versions
- Helps keep team installs consistent across machines
- Reflects the current Expo 54-based dependency graph for this project

## Dependencies Used

- npm lockfile format version 3
- Resolved entries for Expo, React, React Native, TypeScript, and supporting packages

## Notes About Design or Implementation Decisions

- This file is generated by npm rather than edited manually
- It is documented because it is part of the project root and influences reproducible installs, even though `node_modules/` itself is excluded from repository documentation

- Created documentation for the generated npm lockfile after dependency installation
